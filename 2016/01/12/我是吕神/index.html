什么是Hexo？

Hexo的开发者是来自台湾的tommy351大神，是一个相当轻量，简洁且高效的博客框架。
Hexo可以非常快速的生成一个静态博客，只需要通过几行命令即可完成，非常适合非JS前端开发程序猿使用（我JS不NB，但是使用这个框架没有任何难度）。它还有丰富的成熟主题使用，如果你感兴趣，也可以自己编写主题充分展现自己的个性化。
Hexo 使用 Markdown（或其他渲染引擎）解析文章，关于Markdown你可以查看Markdown语法简体中文版。

如何安装Hexo？

安装Hexo十分简单，由于Hexo基于Node.js，并且我们需要部署到GitHub上，所以需要安装以下应用程序:

Node.js
Git

进入官网下载安装程序，安装完后测试，打开控制台，键入“npm”，出现

Usage: npm
表示Node.js已安装成功。同理，键入“git”，来检测Git是否安装成功。
上述环境安装好后，即可使用npm安装Hexo。

npm install -g hexo-cli
如果出现以下错误

npm ERR! Darwin 14.4.0

npm ERR! argv “/usr/local/bin/node” “/usr/local/bin/npm” “install” “-g” “hexo-cli”

npm ERR! node v0.12.1

npm ERR! npm v2.5.1

npm ERR! Attempt to unlock /Users/pokim/npm-global/lib/node_modules/hexo-cli, which hasn’t been locked

npm ERR!

npm ERR! If you need help, you may report this error at:

npm ERR!

npm ERR! Please include the following file with any support request:

npm ERR! /Users/pokim/npm-debug.log
这是因为使用控制台输入该命令时使用的是非管理员身份，’-g‘表示全局安装，需要使用管理员身份。

sudo chown -R $USER /usr/local/lib
该命令是对指定目录下获取root权限，根据提示输入管理员密码（如果你的Mac未设置管理员账户，必须在运行上述命令之前为管理员账户设置密码，具体请看官方文档)，成功后再次运行Hexo安装命令，如果出现

node-gyp rebuild

恭喜你，至此Hexo的环境已安装完毕。

Hexo的建站与配置

建立站点文件夹

Hexo建站也非常的简单，执行下列命令， 初始化包含站点文件的文件夹。

hexo init
cd
npm install
你可以在User文件夹下查看到生成的文件夹，主要目录如下：

.
├── _config.yml #站点配置文件
├── package.json #应用程序的信息
├── scaffolds #模板文件夹，博文是根据模板生成的
├── source #资源文件夹
| ├── _posts #生成的博客
└── themes #下载的主题
现在我们执行下列命令来发布的本地服务器。

hexo g #生成博客静态文件
hexo s #启动本地服务器
在浏览器中打开http://localhost:4000查看默认效果，是不是炫酷的博客已经出现了！不过这还远不能满足我们，我们需要更个性化的定制。

设置Hexo全局配置

打开站点配置文件_config.yml，只需要修改部分配置，大部分保持默认即可。

Hexo Configuration

Docs: http://hexo.io/docs/configuration.html

Source: https://github.com/hexojs/hexo/

Site

title: #博客标题
subtitle: #博客副标题
description: #博客的描述
author: #作者
language: #网站使用的语言，中文一般使用zh-CN
timezone: #时区

URL

If your site is put in a subdirectory, set url as ‘http://yoursite.com/child‘ and root as ‘/child/‘

url: #网站链接，这里我填写的是github的个人页，可以不填
root: /
permalink: :year/:month/:day/:title/
permalink_defaults:

Extensions

Plugins: http://hexo.io/plugins/

Themes: http://hexo.io/themes/

theme: next #主题
使用自己下载的主题时，网站语言可能明明设置zh-CN，但是博客还是没有显示中文简体，这是由于该主题的languages文件中不包含zh-CN，找到该主题下的languages文件夹，查看所包含的中文简体语言文件，比如我现在使用主题的中文就是用的zh-Hans。

写博文

新建博文

由于Hexo使用Markdown，你可以新建.md文件，放在_posts文件夹下，也可以使用以下命令来新建博文。

hexo new “title”
在这里推荐一个Mac的Markdown编辑器Mou，支持双屏浏览，实时保存，不需要手动Ctrl+C保存，否则会有卡顿现象。

Front-matter

打开新建的.md文件，初始内容如下：

title: #标题，默认为文件名相同，实际显示以此为准
date: #创建日期
tags: #所属标签

-–
现在开始使用Markdown写你的博文。
该区域用来定义当前文件的变量，文档说明可以使用如下变量参数：

layout: #布局
title: #标题
date: #建立日期
updated: #更新日期
comments: #开启文章的评论功能，默认开启
tags: #标签
categories: #分类
permalink: #覆盖文章网址
多标签注意语法格式如下：

tags:

- 标签1
- 标签2
- 标签3
- …

你可以添加你所需要的变量，如果觉得每次生成后更改很麻烦，可以在模板文件夹scaffolds下找到post.md文件，编辑它的Front-matter为你想要的变量参数，默认新建博文使用的post模板。

创建制定模板

当然如果你想使用其他模板，你可以使用下列命令来创建指定模板的博文：

hexo new scaffold “title”
请确保使用的模板scaffold在scaffolds存在。

文章摘要

这个功能很实用，可以让你在首页文章目录不必显示全部的文章内容，只显示指定摘要。

以上是摘要
<!–more–>
以下是全文
注意：所有文件配置：后面，都必须留一个空格，都必须留一个空格，都必须留一个空格，重要的事情说三遍。

主题配置

全局变量配置

同学可以去Hexo主题库下载自己喜欢的主题，我目前使用的主题为NexT，基于本主题介绍主题配置，基本上大同小异。
打开相应主题文件夹下的主题配置文件_config.yml：

when running hexo in a subdirectory (e.g. domain.tld/blog), remove leading slashes ( “/archives” -> “archives” )

menu: #菜单配置，页头需要显示哪些栏目
home: / #首页
archives: /archives #归档（文章列表）
categories: /categories #分类（需手动创建）
tags: /tags #标签 （需手动创建）
about: /about #关于（需手动创建）
commonweal: /404.html #丢失页面（需手动创建）

Place your favicon.ico to /source directory.

favicon: /images/favicon.ico #网页图标

Icon fonts

Place your font into next/source/fonts, specify directory-name and font-name here

Avialable: default | linecons | fifty-shades | feather

icon_font: default #字体

#icon_font: fifty-shades

#icon_font: feather

#icon_font: linecons

Schemes

scheme: Mist #该主题下的子主题

Sidebar, available value:

- post expand on posts automatically. Default.

- always expand for all pages automatically

- hide expand only when click on the sidebar toggle icon.

sidebar: post #侧边栏展现方式

#sidebar: always

#sidebar: hide

head image

avatar: http://7xl2f1.com1.z0.glb.clouddn.com/head.jpg #侧边栏头像
menu下需要手动创建的栏目，可以通过执行以下命令：

hexo new page “page”
如果不需要启用Mist主题，在前面使用#注销即可。

第三方服务

评论与分享

这里推荐使用多说，注册账号后，在首页选择“我要安装”，创建站点，填写信息。
打开站点配置文件（不是主题配置文件），增加如下配置：

#Duoshuo
duoshuo_shortname: name #唯一标示
duoshuo_share: true #是否开启分享
duoshuo_hotartical: true #是否开启热评
其中表单中的多说域名就是需要填写的duoshuo_shortname，这样就可以通过多说来进行分享，回复与热评了。

站点统计

国内推荐使用百度统计，注册百度统计，添加博客地址，然后进入查看代码界面：

打开站点配置文件（不是主题配置文件），增加如下配置：

#Baidu
baidu_analytics: code
code填写图中红色区域你自己的代码，这样你就可以通过百度统计来查看对于自己博客的各项站点分析了。
关于NexT主题以及更多第三方服务请参考NexT官方文档。

如何部署到GitHub？

站点配置

部署到GitHub上其实也非常简单，打开站点配置文件_config.yml：

Deployment

Docs: http://hexo.io/docs/deployment.html

deploy:
type: github #部署工具的类型
repository: git@github.com:username/username.github.io.git #SSH链接
branch: master #分支
SSH密钥

这里我们需要使用到SSH来建立你的Mac和GitHub的联系，而不能使用一般的HTTP的URL，因为部署的操作需要上传到服务器，所以需要一个安全可信赖的环境，所以这里需要使用SSH。
打开控制台，输入以下命令建立新的SSH密钥：

ssh-keygen -t rsa -C “username@example.com”
这里使用你提供的邮箱创建SSH秘钥，接着出现以下信息：

Generating public/private rsa key pair.
Enter file in which to save the key (/Users/you/.ssh/id_rsa): #这里直接enter确认
Enter passphrase (empty for no passphrase): #输入密码，建议非空
Enter same passphrase again: #确认密码
完成之后，会有以下提示：

Your identification has been saved in /Users/you/.ssh/id_rsa.
Your public key has been saved in /Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
keys username@example.com
keys是一组钥匙串信息，这里我们就把SSH密钥创建完毕了。

添加SSH Keys到Github

如果没有GitHub账号，请前往GitHub官网注册账号，不会注册的请破腹自尽- -。
登录成功后，点击右上角账户选择打开setting，选择SSH keys，如图所示：

这里我们点击Add SSH Key，然后输入标题（随意），Key需要输入我们先前创建好的密钥内容，打开控制台，键入以下命令复制密钥内容：

pbcopy < ~/.ssh/id_rsa.pub
然后粘贴至Key栏，点击Add key，提示确认密码，输入你的账户密码就完成了。
然后我们键入如下命令检测是否成功连接本机与GitHub：

ssh -T git@github.com
如果是第一次使用，会显示以下警告，提示你是否确认连接：

The authenticity of host ‘github.com (207.97.227.239)’ can’t be established.
RSA key fingerprint is keys.
Are you sure you want to continue connecting (yes/no)?
输入yes确认即可，如果提示以下信息：

Hi username! You’ve successfully authenticated, but GitHub does not provide shell access.
☝️完美☝️！

创建GitHub Pages

我们托管静态博客，使用的就是GitHub Pages的功能，首先我们需要创建一个个人主页，点击右上角的加号，选择new repositories，进入创建页面

这样我们以后可以通过username.github.io来访问你的主页，点击Create repository创建成功，然后点击侧边栏的setting，进入设置页面：

点击设置成主页，跳转后无需填写信息直接点击Continue to Layouts为主页选择一个主题：

点击publish pages就创建个人主页成功了，这也是我们托管自己博客的仓库，你可以通过username.github.io来先浏览刚生成的个人主页。

部署到GitHub

打开站点配置文件_config.yml，在需要填写SSH的地方填入git@github.com:username/username.github.com，保存后打开控制台键入以下命令：

cd blog #进入博客站点文件夹
hexo g #生成静态网页
hexo d #部署
如果出现以下信息：

INFO Deploying: git
INFO Clearing .deploy folder…
INFO Copying files from public folder…
……..
INFO Deploy done: git
恭喜你，博客已经部署成功，快通过username.github.io访问你的博客吧！
